2. Installation

2.1 How to obtain Kscd

Kscd is an application for the KDE project http://www.kde.org. Kscd can be found on ftp://ftp.kde.org/pub/kde/, the main ftp site of the KDE project.

2.2 Requirements

In order to successfully compile Kscd, you need the latest versions of libkdecore and libkdeui. All required libraries as well as Kscd itself can be found on ftp://ftp.kde.org/pub/kde/.

2.3 Compilation and installation

In order to compile and install Kscd on your system, type the following in the base directory of the Kscd distribution: 

% ./configure

NOW YOU MUST CHECK THE FIRST FEW LINES OF THE plat_xxx.c FILE CORRESPONDING TO YOUR PLATFORM AND MAKE THE NECESSARY CHANGES.

I apologize for the inconvenience, but a cdrom player is a device driver of sorts and it very hard if not impossible to find out what sort of hardware you have. I hope to be able to provide an interactive installation script in the near future.

I have made changes to the original workman cd placer source code that made kscd the fastest, most responsive and quietest cd-player for an average Linux IDE cdrom drive. I hope you will find this to hold true for your particular hardware too. In the unlikely case that my changes do not work for your particular hardware setup, setting the proper defines will disable those changes and revert to the original workman behavior.

I have also added code from Dirk Foersterling which should give you better playing if you use the sbpcd or mcdx drivers under Linux. You must however explicitely enable this by defining the appropriate defines in plat_linux.c.

If you are experiencing trouble getting kscd to work on a non-Linux platform please study the original Workman documentation in the docs directory of the kscd distribution. They contain quite a bit of information and may prove to be very helpful. The original makefiles are in the makefiles subdirectory and should also be of interest in case of trouble.

Please inform me of any modification you had to undertake to the current distribution in order to get things to work on your platform. I will then include those changes in future versions of kscd.

% make
% make install

Again: Should you run into problems, please report them to me.
